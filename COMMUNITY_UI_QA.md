# ROOTED Community — UI QA Checklist

This checklist is vertical-specific and does not override platform law.

---

## 1. Discovery UI

MUST:
- Use backend-only results  
- Enforce 6–8 card rows  
- Apply specialty filters correctly  
- Respect GEO boundaries ≤ 50 miles  
- Rotate providers based on backend order  

MUST NOT:
- Show rankings  
- Show ads  
- Show pricing in Kids Mode  

---

## 2. Vendor Profiles

Show:
- Photos  
- Story  
- Hours  
- Specialty  
- Location  
- Gallery  

Hide:
- Analytics (reserved for premium verticals)  
- B2B tools  
- RFQ/bid/bulk buttons  

---

## 3. Events UI

- Must show family-friendly categorization  
- Volunteer events must hide unsafe activities from minors  
- Kids Mode removes price fields  

---

## 4. Landmarks

Kids Mode:
- Only parks, farms, nature zones, educational sites  

Normal Mode:
- All community-safe landmarks  

---

## 5. Map UI

- Max 25 markers  
- Max 50-mile radius  
- Kids Mode shrinks categories  

---

END OF COMMUNITY QA
